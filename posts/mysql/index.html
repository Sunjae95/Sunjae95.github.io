Mysql

Dec 30, 2020 2020-12-30T21:30:00+09:00 by SeonjaeLee
2 min

세계에서 가장 많이 쓰이는 오픈소스의 관계형 데이터베이스 관리 시스템(RDBMS)이다.
다중 스레드, 다중 사용자 형식의 구조질의어 형식의 데이터베이스 관리 시스템이다.
MYSQL을 사용하는 이유는 내가 회원가입 및 자신이 올린 글 등록, 삭제 및 수정 기능을 구현하기 위해 사용하였다.

내가하는 방식은 form에 있는 데이터들을 POST방식으로 전달하여 BODY내용을 MYSQL을 통해 저장하고 보여주는 방식이다.

require('dotenv').config({path: './.env'});
 

// .env
DB_HOST=localhost
DB_USER=root
DB_PASS=password
DATABASE=dbname
DB_PORT=3306


// ./db/index.js
const mysql = require('mysql2/promise');

const env = process.env;

const pool = mysql.createPool({
    host: env.DB_HOST,
    user: env.DB_USER,
    password: env.DB_PASS,
    database: env.DATABASE,
    port: env.DB_PORT,
    connectionLimit: 10,
});

module.exports = pool;


// ./routes/register.js
const express = require('express');
const router = express.Router();
const db = require('../db');


router.post('/', async (req, res) => {
    const user = req.body; // -> { id: "adf", password: "123" }

    const [ rows ] = await db.query('INSERT INTO user SET ?', [ user ]);
    console.log(rows.insertId);
    
    res.send('ok');
});

    app.js는 서버를 실행해주는 파일이다. 이 파일을 언급한 이유는 env파일때문이다.
    env파일을 사용하는 이유는 개인정보 및 MYSQL의 정보를 보호하기 위함이다.

    .env는 JSON파일 형식과 비슷하나 ‘‘이나 ““없이 NAME=NAME형식으로 작성하면된다.

    db/index.js는 creatPool를 생성하여 DB에 접근할수있도록 모듈화 해놓은 파일이다.

    routes/register.js는 db변수에 모듈화된 pool을 참조한다.
    post방식으로 body내용을 user에 저장하고 DB쿼리문을 사용하여 데이터베이스에 데이터를 추가해준다.
