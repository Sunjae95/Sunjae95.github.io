# 체육복
![문제](/assets/img/algorithm/체육복.PNG)

# my solution
greedy 알고리즘 문제는 해결하는 과정에서 그 순간순간마다 최적이라고 생각되는 결정을 하는 방식으로 진행하여 최종 해답에 도달하는 문제 해결 방식이다.  

나는 이 문제를 풀기 위해 다음과 같이 생각했다.
1. 여벌 가져온 학생(`reserve`)이 도난을 맞았을 때(`lost`) 제외시킨다.
2. 여벌을 가져온 학생이 도난 맞은 학생 빌려주기

```javascript
function solution(n, lost, reserve) {
    let answer=n-lost.length;
    //여벌가져온 학생이 도난 맞았을때 
    for(let i=0; i<lost.length; i++) {   
        for(let j=0; j<reserve.length; j++) {
            if(lost[i]>0 && reserve[j]>0){  //탐색조건
                if(lost[i] === reserve[j]) {
                    lost[i]=-1;
                    reserve[j]=-1;
                    answer++;
                }
            }  
         }
    }
    //여벌 가져온 학생이 도난 맞은 학생 빌려주기
    for(let i=0; i<lost.length; i++) {
        for(let j=0; j<reserve.length; j++) {
            if(lost[i]>0 && reserve[j]>0) { //탐색조건
                if(lost[i] === reserve[j]-1) {
                    lost[i]=-1;
                    answer++;
                }else if(lost[i] === reserve[j]+1) {
                    lost[i]=-1;
                    answer++;
                }
            }
        }
    }
    return answer;
}
```

처음에는 내가 생각한 1번 조건을 넣지않아 정답률이 50%대 였는데, 제한사항을 읽어보니 1번을 생각하게 되어 1번과 2번을 생각하게 되었다.  
하지만 for문을 돌릴 때 중복되어 answer이 증가하는 것을 확인하게 됐다.  
그래서 for문에 탐색조건을 넣어 풀었다.  
하지만 결과가 75점이라 다른 사람의 풀이를 참고하였다.  
## 다른 풀이 
```javascript
function solution(n, lost, reserve) {
    const arr = {};
    let answer = 0;
    for(let i = 1; i <= n; i++){
        arr[i] = 1;
    }
    lost.forEach(number => arr[number] -= 1);
    reserve.forEach(number => arr[number] += 1);

    for(let i = 1; i <= n; i++){
        if(arr[i] === 2 && arr[i-1] === 0){
                arr[i-1]++;
                arr[i]--;
        } else if(arr[i] === 2 && arr[i+1] === 0){
                arr[i+1]++;
                arr[i]--;
        }
    }
    for(let key in arr){
        if(arr[key] > 0){
            answer++;
        }
    }
    return answer;
}
```
나는 answer의 숫자를 갱신하는 방법으로 하였고 다른 풀이는 따로 object를 만들어 나중에 answer을 센 거 같다.  
나도 처음에는 따로 배열을 만들어 나중에 answer을 카운트하는 방향으로 잡았지만 그 당시에는 java를 하다가 javascript로 코딩을 하려고 하니 어떻게 하는지 몰라서 어려움을 겪어 내 방식 대로 하였다.  
javascript object나 배열을 공부를 하고 문제를 풀어보면 훨씬 더 좋을 거 같다고 느꼈다.
